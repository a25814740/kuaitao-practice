# 塊陶啊!!｜Vue 3 求職練功專案規格

## 1. 專案目的
這是一個用來練習 Vue 3 的 side project。

使用者背景：
- 有 8 年 HTML / CSS / JavaScript / jQuery / Vue 2 基礎
- 約 2 年未實際開發，想重新找回手感
- 求職目標以 Vue 3 職缺為主
- 不想用純看影片的方式學習，希望透過「做出有趣作品」學 Vue 3
- 希望能從簡單到困難，逐步完成一個完整 App
- 每個階段都要看得到學習進度、成果差異，方便之後複習

此專案不是一開始就追求完整商業化，而是以「學習路徑清楚、每階段可驗收、適合作品集展示」為優先。

---

## 2. 專案概念
專案名稱暫定：**塊陶啊!!**

概念：
- 類似「薪水小偷」的情緒型 App
- 核心不是摸魚，而是記錄「下班那一刻」
- 使用者可以記錄下班、加班、排行、好友互動、貼文抱怨
- 台灣市場可用「塊陶啊!!」這個諧音梗
- 其他中文市場可替換為「快逃啊」

產品核心價值：
- 儀式感：下班那一瞬間變成可記錄、可炫耀的事件
- 情緒共鳴：對加班文化的幽默反抗
- 遊戲化：排行、成就、稱號、等級、任務
- 社交性：好友互比、貼文互動、行業/地區排行

### 中期版本

- 留言互動

### 長期版本

## 4. 核心功能清單

### 4.1 使用者初始化設定
第一次進入 App 需先設定：
- 暱稱
- 大頭貼
- 工作類型（先固定幾種）
- 每天下班時間
- 收入

備註：
- 工作類型先寫死：例如工程師、設計師、行銷、客服、餐飲、行政、公職、醫療、其他
- 大頭貼一開始可先用預設圖或 emoji / 頭像字母
- 收入後續可用於推導時薪、加班感受、戰績數據

### 4.2 首頁
- 顯示目前時間
- 顯示距離下班還有多久
- 顯示今日狀態（未到下班 / 已到下班 / 已加班）
- 顯示一個主按鈕：下班點我 / 塊陶啊!!
- 點擊後記錄當下時間
- 記錄後顯示結果卡片

### 4.3 下班記錄
- 記錄按下按鈕的實際時間
- 顯示與預設下班時間差距
- 可判斷：
  - 準時下班
  - 提早下班
  - 延後下班
- MVP 先精確到毫秒即可
- UI 可以營造微秒感，但後端與資料記錄不必真的追求奈秒

### 4.4 排行榜
- 可先用假資料 / 本地資料實作 UI 與排序邏輯
- 分類：
  - 好友
  - 全國
  - 地區
  - 行業
- 之後再串 Supabase

### 4.5 社交牆
- 使用者可發簡短抱怨 / 心得
- 其他人可留言
- MVP 可先做假資料版
- 第二階段再做真實資料庫版

### 4.6 遊戲化
- 等級
- 成就
- 稱號
- 任務

成就範例：
- 連續 7 天準時下班
- 連續 3 天加班超過 2 小時
- 第一次在下班時間後 1 秒內點擊

## 6. 分階段學習與開發路線

> 核心原則：每一階段都要「能跑、有成果、可回顧、可展示」。

- `ref`
- 事件綁定
- 基本模板語法

目標：

### Stage 2｜條件渲染與計算邏輯

## 7. 時間精度設計原則

### UI 層
- 可以顯示到毫秒
- 可以用動畫營造微秒感
- 用 `performance.now()` 增強畫面表現

### 資料層
- MVP 只需記錄到毫秒即可
- 因為網路延遲與裝置時間差，真正追求微秒或奈秒在產品上意義不大
- 後端儲存重點是「使用者按下當下的 client timestamp」與「預設下班時間差值」

結論：
- 微秒主要是儀式感與 UX
- 不必一開始就把資料庫與 API 設計成高精密科學系統

## 9. 給 Codex 的執行原則

Codex 在此專案中應遵守：

1. 先以「學習型專案」為目標，不要一開始過度工程化。
2. 優先讓每個 stage 都有可執行成果。
3. 每完成一個 stage，必須更新：
   - `docs/STAGE_CHECKLIST.md`
   - `docs/DEVLOG.md`
4. 每次修改都要說明：
   - 本次學到的 Vue 3 概念
   - 與前一階段差異
   - 本階段完成內容
   - 下一階段建議
5. 優先保留簡潔架構，不要太早引入不必要的抽象。
6. 如果需求尚未進入 Supabase 階段，就先用本地假資料。
7. UI 可先簡潔，但需具備作品集展示感。
8. 優先使用 TypeScript 與否可視使用者當前學習負擔決定；若會增加太多阻力，可先用 JavaScript 版本完成，再逐步轉型。

## 12. 後續給 Codex 的工作方式

之後每次只下達一種指令：

### 模式 A：請做下一階段

### 模式 B：請複習目前專案
> 請根據目前專案與 docs/DEVLOG.md，整理我到目前為止學到的 Vue 3 概念，列出每一階段做了什麼、為什麼要這樣拆、哪些地方是面試重點。

> 請在不跳過當前學習階段的前提下，改善目前程式碼結構與命名，保留學習路徑的可理解性，並更新 DEVLOG。
